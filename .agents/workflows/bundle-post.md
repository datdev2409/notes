# Workflow: /bundle-post [filename]

When the user invokes `/bundle-post [filename]` (where filename is a file in the `drafts/` directory), follow these steps:

1. **Read Draft**: Use `view_file` to read the contents of `drafts/[filename]`. If no filename is provided, use `list_dir` on `drafts/` and ask the user which draft they want to bundle.
2. **Create Leaf Bundle Folder**: Determine a SEO-friendly slug for the post based on its content. Ensure the folder follows the Leaf Bundle format: `content/posts/YYYY-MM-DD-[slug]/`.
3. **Generate YAML Front Matter**: Analyze the draft content and generate standard Hugo YAML front matter variables: 
   ```yaml
   ---
   title: "Autogenerated Title"
   date: YYYY-MM-DD
   tags: ["tag1", "tag2"]
   categories: ["cat1"]
   description: "Autogenerated concise description"
   draft: true
   ---
   ```
4. **Process Content & Format Markdown**: 
    - Format the draft beautifully. Ensure all code blocks have language identifiers (```bash, ```yaml).
    - Add GitHub Markdown Alerts (`> [!NOTE]`, `> [!TIP]`, `> [!WARNING]`) where appropriate to emphasize important points.
    - If there are images referenced in the draft, rewrite the Markdown paths to relative paths `![Alt Text](image_name.png)`. 
5. **Moving Images (If Applicable)**: 
    - If you notice image files in the `drafts/` directory that belong to this post, use `run_command` with `mv` to move them into `content/posts/YYYY-MM-DD-[slug]/`.
6. **Write Final Post**: Use `write_to_file` to create `content/posts/YYYY-MM-DD-[slug]/index.md` containing the front matter and processed content.
7. **Suggest Enhancements**: If the content describes an architecture, flow, or complex relationship, suggest creating a Mermaid diagram directly in the post. Wait for user confirmation.
8. **Cleanup**: Ask the user if they want you to delete the original draft file in `drafts/`.
